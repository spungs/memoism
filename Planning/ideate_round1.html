아이디어 발산 (총 22개)

전체 Quick Win Big Bet (베타 필수) V2 미룸 폐기

HMW 1 마찰 · 1분 일기

H1.1QUICK WIN
단일 도발 질문
AI가 사진 5장 분석 후 가장 핵심적인 질문 1개만 생성 → 답변 1줄 → 일기 완성.
H1.2고려
3-tap 일기
사진 업로드 → 감정 1탭 → 한 줄 → 저장. 인터랙션 3번으로 끝.
H1.3폐기
선다형 답변
AI가 4지선다 객관식. T1의 정확성("내 진실") 가치와 충돌.
H1.4V2
음성 1초 답변
마이크 누르고 1~10초 발화 → STT → 자동 저장. 운전·잠자리 시점 fit이 명확하나 (D) 텍스트로 먼저 검증.
H1.5폐기
사진 캡션만
5장 사진에 한 줄씩, 본문 X. USP("회고할 데이터")가 약해짐.
H1.6QUICK WIN
Skip 안전망
Q&A 답변 안 해도 사진만 저장 OK. "회고할 데이터"는 사진만으로도 충족.
H1.7폐기
누적 합성
같은 날 짧게 여러 번 입력 → 자동 합성. 베타엔 복잡.

HMW 2 정확성 · 환각 0

H2.1⭐ 베타 필수
AI는 질문만, 본문은 합성
AI가 생성하는 건 질문뿐. 일기 본문 = EXIF 사실 + Vision 명사구 + 사용자 답변. AI는 생성기가 아니라 합성기. 베타의 구조적 척추.
H2.2QUICK WIN
Fact/Story 시각 분리
일기 카드에 사실 영역(시간·장소·사진)과 이야기 영역(사용자 답변) 시각 구분.
H2.3QUICK WIN
AI 라벨 (보조)
AI가 쓴 부분에 "✨" 마커. H2.2와 통합 가능.
H2.4⭐ 베타 필수
Vision 응답을 명사구로 제한
"강아지를 쓰다듬는 즐거운 시간" ✗ / "강아지 한 마리, 거실, 낮" ✓. 해석 금지, 사실 명사구만. H2.1과 한 쌍.
H2.5QUICK WIN
사용자 검토 게이트
자동 생성 일기는 반드시 사용자 컨펌 후 저장. 1초의 "이대로 저장" 강제.
H2.6QUICK WIN
EXIF 헤더 우선 표기
일기 상단에 EXIF 시간/장소. "사실이 1순위" 시각 신호.
H2.7폐기
"AI 추측" 토글
디폴트 OFF. H2.1로 구조적 해결, 토글 불필요.

HMW 5 검색·공유 · 3초 회상

H5.1⭐ 베타 필수
자연어 검색 (RAG)
"작년 봄 카페" → 임베딩 매칭 top-5 일기. pgvector 인프라 필요.
H5.2폐기
음성 검색
마이크 탭 → 발화 → STT. H1.4와 묶어 V2.
H5.3QUICK WIN
결과 = 사진 그리드
검색 결과를 일기 카드가 아닌 사진 그리드로 표시. 같이 보기 좋게.
H5.4V2
공유 모드 (가림)
"함께 보기" 버튼 → 사진+한줄 풀스크린, 일기 본문(사적)은 자동 가림. GTM 트리거.
H5.5폐기
공유 카드 이미지 생성
일기 → PNG 카드. 디자인·렌더링 비용 큼. OS share만으로 베타 충분.
H5.6폐기
컨텍스트 검색 (캘린더)
캘린더 권한·privacy 이슈. 베타 범위 밖.
H5.7V2
Passive 자동 추천
홈에 "오늘 분위기와 비슷한 과거 일기" 카드. HMW 3(회고)과 함께 v2.
H5.8폐기
타임라인 모드
검색 결과 시간순 가로 스와이프. H5.3 사진 그리드로 충분.
