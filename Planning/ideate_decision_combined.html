🔥 Big Bets — 베타 척추 3

F1. AI 본문 자동 생성·정리 (모드 A·B·C) SPINE
사진 5장 + EXIF / 사용자 텍스트 / 둘 다 → AI가 1인칭 150~250자 일기 본문 생성·정리. AI는 사진에 없는 사실·사용자가 안 쓴 의미를 추가하지 않음 (환각 0 정책).
입력 사진 1~5장 OR 텍스트 출력 1인칭 150~250자 모델 Gemini 2.5 Flash 멀티모달 비용 약 ₩1/회
F2. 사용자 검토 게이트 SPINE
자동 생성된 일기는 *반드시* 사용자가 검토 후 저장. 수정·승인·재생성 가능. EXIF 시간·장소는 헤더에 고정 표기, AI 생성 본문은 옅은 배경색으로 시각 강조.
플로우 생성 → 검토 → 수정/승인/재생성 완료 기준 사용자 승인 = 저장 트리거
F3. 자연어 검색 (RAG) SPINE
"작년 봄 카페" 같은 자연어 → 임베딩 매칭 top-5 일기. Basic+ 사용자는 캐릭터와 채팅 인터페이스. 무료 사용자는 키워드 검색만.
인프라 pgvector + Gemini text-embedding-004 비용 ₩0.1/회 미만 접근 Basic+ (무료는 키워드)

⭐ Quick Wins 6

F4. Skip / 빠른 승인QUICK
텍스트·사진 없이도 사진만 저장 OK. 검토 없이 즉시 승인도 OK. "회고할 데이터" 명제는 사진만으로도 충족.
F5. Fact/Story 시각 분리 + AI 라벨QUICK
일기 카드 = 사실 영역(EXIF 시간·장소·사진) + 이야기 영역(본문). AI 생성 부분에 ✨ 마커.
F6. EXIF 헤더 우선 표기QUICK
일기 상단에 촬영 시간·장소 자동 표기. "사실이 1순위" 시각 신호.
F7. 검색 결과 사진 그리드QUICK
검색 결과를 일기 카드가 아닌 사진 그리드로 표시. 친구·연인과 같이 보기 좋게.
F8. 본문 톤·길이 정책QUICK
1인칭 + 150~250자 (베타 고정). V2에서 페르소나 프리셋·커스텀 추가.
F9. 트리거 조건QUICK
AI 호출은 사진 1장+ OR 텍스트 비어있지 않을 때 활성. 둘 다 비면 버튼 비활성.

⑩ Multi-Agent 기능 설계

베타는 단일 LLM 호출 + 모드 분기 구조. V2 이후 에이전트 분리 검토.

베타 — 단일 엔진, 모드 분기

입력
사진(0~5장) + 텍스트(0~?자) + 페르소나(고정)
분기
사진만 → 모드 A / 텍스트만 → 모드 B / 둘 다 → 모드 C
처리
Gemini 2.5 Flash 멀티모달 단일 호출. 모드별 시스템 프롬프트 차이.
출력
JSON { title, content, suggestedMood } → Zod 검증 → 사용자 검토 게이트
백그라운드
저장 후 임베딩 자동 생성 (text-embedding-004)

V2 — 에이전트 분리 후보

1. Diary Writer Agent

사진/텍스트 → 일기 본문 생성·정리. 페르소나·톤·길이 적용. 현재 베타의 단일 엔진을 분리.

2. Memory Companion Agent (캐릭터)

RAG 검색 + 대화. 사용자 일기 컨텍스트 유지. Basic+ 한정. 일기를 "다 본 친구" 페르소나.

⑪ 정책·엣지케이스

같은 날 사진 — B-1.smart

  • EXIF 있는 사진들끼리 = 같은 KST 날짜여야 함 (다르면 reject)
  • EXIF 없는 사진은 "정보 없음" 마커 + 사용자 입력 날짜 사용
  • 자정 걸침 (23:55~00:15): 가장 많은 사진이 속한 날짜 자동 선택 + 한 줄 안내
  • 해외 사진 timezone 누락: KST로 해석 (V2에서 timezone 처리)
